Class
Given the sets \( T_1, T_2, T_3, ..., T_n \), then the notation 
                class C:
                    a1: T1
                    a2: T2
                    a3: T3
                    ...
                    an: Tn

Defines the function \( C : \prod _ { j = 1 } ^ n T_j \to \mathcal{O} \), where \( \mathcal{ O } \) is said to be the set of objects

The objects are defined as the functions of the form \( O: \left\{ \texttt{a_1}, \texttt{a_2}, \ldots, \texttt{a_n} \right\} \to \bigcup _ { i = 1 } ^ n T_i \), such that for any \( i \in [n] \) we have \( O \left( \texttt{a_i} \right) \in T_i \)

With this in place, we define that if \( O = C \left( \textbf{t} \right) \) and \( \textbf{t} \in \prod _ { i = 1 } ^ n T_i \), then \( O \left( \texttt{a_i} \right) = t_i \).

We then give this object a special syntax, given any object, we define the following notation \[ O.\texttt{a_i} := O \left( \texttt{a_i} \right) \]

While this definition seems complicated, it's less complicated then it looks, for example consider the following class definition, where \( S \) is the set of all strings. 

        class Car:
            model: S
            num_wheels: N

Then suppose we set \( \textrm{my_car} := Car \left( \texttt{Honda}, 4 \right) \), then \( \textrm{ my_car.model } = \texttt{Honda} \) is true.
